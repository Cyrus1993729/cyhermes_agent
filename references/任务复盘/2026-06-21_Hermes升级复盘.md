# 复盘：Hermes Agent 0.14.0 → 0.17.0 升级

### 📋 任务标识
- **日期**：2026-06-21
- **任务名称**：Hermes Agent 跨三大版本升级
- **耗时**：约 6 小时

### 🎯 任务目标
将 Hermes Agent 从 0.14.0 升级到 0.17.0，保留全部配置、Skill 和 API Key。同时升级 Python 3.11→3.12。

### 🔄 实际流程
1. Python venv 切换（3.11→3.12）→ 中途被 gateway 中断打断 3 次
2. 改为写 .bat 脚本让用户手动执行 → 5 轮迭代修复
3. 编码问题（中文乱码）→ 全改成英文
4. Claude 两轮审查 → 修复 7 个 bug
5. Git 仓库损坏 → 重建 .git + shallow fetch
6. 最终成功升级到 0.17.0，但 Python 被回退到 3.11

### ⚠️ 踩过的坑
1. **gateway 中断死循环**：停 gateway 的命令恰好需要走微信通道 → 停了就联系不上。解决：写脚本让用户手动执行
2. **中文批处理编码**：GBK vs UTF-8 导致乱码被当命令执行。解决：全部改成英文
3. **git checkout -B 不生效**：本地仓库无 commit，checkout 没切到远端最新。解决：rm .git + fetch --depth 1
4. **start 命令嵌套引号**：路径含空格时窗口闪退无声失败。解决：去除嵌套引号
5. **Python 3.12 venv 被回退**：git checkout -f 可能覆盖了 venv。暂时接受 3.11

### ✅ 最终结果
- Hermes 成功升级到 0.17.0 ✅
- 所有配置、Skill、API Key 保留 ✅
- 桌面客户端已安装 ✅
- Python 仍为 3.11.15 ⚠️（已知 Bug #42810 暂未触发）

### 🔧 可优化点
1. 升级脚本应该在一开始就写成 .bat 让用户手动执行，避免反复 gateway 中断
2. 应该先验证 git 仓库完整性再开始升级
3. 重大环境变更（Python 版本）应该在升级前 snapshot 验证
