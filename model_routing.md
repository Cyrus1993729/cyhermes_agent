# 模型路由速查表 — Model Routing

> 什么任务用什么模型。决策依据：任务类型 × 复杂度 × 成本。
> 每次路由决策后记录，供复盘优化。

---

## 可用模型

| 模型 | 调用方式 | 成本 | 速度 | 定位 |
|:---|:---|:---|:---|:---|
| DeepSeek v4-pro | 默认 provider | 低 | 快 | 主力：日常对话、分析、写作 |
| 千问 3.7 Max | qwen-bailian API | 低 | 快 | L1 审查（自动固化） |
| MoA china | moa preset | 中 | 慢 | 多角度探索、Opus 前置筛选 |
| Opus | Claude Code CLI | 高 | 慢 | 终审：深度推理、最终决策 |

---

## 路由规则

### 🟢 DeepSeek（默认，无需判断）
- 日常对话、信息查询、文件操作
- 中等复杂度的分析（小红书/B站内容分析）
- **规则**：没命中下面的特殊规则时，一律走 DeepSeek

### 🔵 千问 L1 审查（自动）
- 所有交付物型任务收尾
- 已固化在 `qwen_review.py`，不需手动判断

### 🟡 MoA（前置探索，不替代决策）

MoA 比 Opus 便宜但推理深度不如 Opus。定位：**在你不确定要不要花 Opus 的钱时，先用 MoA 探路，拿到多角度分析后再决定**。

**什么时候用 MoA**：
- 问题复杂，但不确定值不值得叫 Opus → 先 MoA 看多角度分析
- 需要"从不同角度分析同一个问题"
- MoA 结果出来后 → 你判断是否还需要 Opus 深挖

**什么时候不用 MoA**：
- 你确定要让 Opus 看 → 直接 Opus，别绕路
- 简单问题 → DeepSeek 足够

### 🔴 Opus（终审，永不自动触发）
- 架构设计审查、论文深度解读、系统性方案设计
- 投资决策相关（涉及真金白银的结论）
- 用户说"让 Opus 看看"
- **规则**：Opus 只在以下情况触发：①L1 审查建议升级且你确认 ②你主动要求。绝不自动触发。

---

## 决策流程

```
接到任务
  ├─ 纯聊天/查信息 → DeepSeek
  ├─ 交付物型任务
  │   ├─ 先做 → DeepSeek
  │   └─ 做完 → 千问 L1（自动）
  │       └─ L1 建议升级 + 你确认 → Opus
  ├─ 复杂/多角度分析
  │   ├─ 不确定要不要 Opus → MoA 探路 → 你判断是否升 Opus
  │   └─ 你确定要 Opus → 直接 Opus
  └─ 你说"用 MoA/Opus" → 按你说的来
```
