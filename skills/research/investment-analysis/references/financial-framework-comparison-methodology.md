# 金融框架对比方法论

当用户要求对比不同理财专家/机构/国家的储蓄率、资产配置比例或投资建议时，必须遵循以下原则。

## 核心原则

### 1. 从零开始，不带结论
- 不要沿用之前对话中得出的数字作为"锚"
- 每次对比都要独立验证
- 用户说"忘掉之前得出的结论，从0开始探讨"时立刻照做

### 2. 统一标准后再对比
不同框架的"储蓄率"含义完全不同：
- 有的含应急金+还债+一切储蓄
- 有的仅退休金
- 有的按税前、有的按税后
- 有的含雇主匹配、有的不含

**必须做的事：**
1. 逐项拆开每个框架的组成（应急金/短期目标/长期退休/住房/医疗）
2. 只提取"纯锁长期权益投资"的部分
3. 统一折算为"占税后可支配收入百分比"
4. 用统一标准表呈现

### 3. 逐个核实口径
- 每个案例必须独立验证其"口径"是否与对比目标一致
- 分为三类：✅能确认口径一致 / ⚠️有数据但口径有差异 / ❌不能确认
- 不能确认的案例不要放进最终对比，另列说明原因
- 不要因为想凑案例数量而放松标准

### 4. 不要硬推结论
- 用户说"我不需要你硬往哪个方向论证"时，停止推动特定结论
- 不要为了证明之前算出的数字合理而选择性引用
- 呈现所有案例后，让用户自己判断
- 用户可以接受"这个范围内，不同专家建议不同"的结论

### 5. 区分"找案例"和"得结论"
- 第一阶段：只找案例，不评价哪个对
- 第二阶段：统一标准，不偷换概念
- 第三阶段：用户自己消化后决定方向

## 反面教材（本会话中的错误）

1. **带着结论找证据**：之前算出¥7-7.5万/年后，下意识寻找能支撑这个数字的案例
2. **口径混淆**：把沃伦的"20%含应急+还债"和拉姆齐的"15%纯退休"放在同张表里直接比
3. **忽略用户纠正**：用户多次指出"20%不是纯锁死比例"后才修正
4. **结论导向**：在用户明确说"不要硬推结论"后，仍然试图构建论证链

## 正确流程

```
第1步：找到 N 个有权威来源的案例
第2步：逐个拆开每个案例的组成（什么被算进去、什么没算）
第3步：只保留口径一致的案例，口径不同的单独标注
第4步：统一折算为同一标准（如"占税后可支配收入%"）
第5步：列表呈现，不评价孰优孰劣
第6步：让用户自己判断适用于自身情况的范围
```

## 常见口径差异速查

| 表面看起来 | 实际可能包含 | 需要剔除 |
|-----------|-------------|---------|
| "储蓄率 20%" | 应急金+短期+长期+还债 | 只留长期退休部分 |
| "退休储蓄 15%" | 含雇主匹配 | 只取雇员部分 |
| "CPF 37%" | 住房+医疗+养老 | 只取 Special Account |
| "百万富翁储蓄率 17%" | 含房产净值+生意投入 | 只留金融资产 |
