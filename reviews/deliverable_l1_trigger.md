# 方案：用户兜底

## 结论
在 Hermes 当前架构下（无 Stop hook / 无输出拦截器），无法通过 prompt/memory/规则 确保 Agent 100% 在交付物完成后触发 L1 审查。

## 原因
LLM 在完成交付物后的最强本能是"交还控制权、问下一步"（turn-end attractor）。任何放在 memory 里的规则都活在模型的注意力预算中，无法稳定对抗这个本能。这是物理限制，不是规则写得不够好。

## 方案
用户作为兜底机制。当用户发现 Agent 未触发 L1 审查时，手动提醒触发。

## 论证
- 优点：零开发成本、零延迟、100% 可靠（用户是最精确的审查触发判断者）
- 缺点：依赖用户记忆和注意力
- 替代方案探索过但不可行：
  - Opus 方案（Stop hook）→ Hermes 不支持
  - 千问方案（输出拦截器）→ Hermes 不支持
  - Opus 备选（cron 轮询）→ 用户否决，2 小时延迟不可接受

## 现阶段决策
维持用户兜底。等 Hermes 支持 guardrail/输出拦截功能后再自动化。
